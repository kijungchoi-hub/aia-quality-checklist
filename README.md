# AIA Quality Checklist

AI 품질 점검 체크리스트와 관련 산출물을 관리하는 저장소입니다.  
현재 저장소에는 답변 품질, 요약 품질, 범용 AI RAG 서비스 품질 점검 문서와 이를 표·트래커 형태로 가공한 결과물이 포함되어 있습니다.

## 목적

- 원본 품질 점검 데이터를 정리하고 재사용 가능한 체크리스트로 구조화
- 답변 품질과 요약 품질 개선 항목을 최종 문서로 정리
- AI RAG 서비스 전반에 적용 가능한 범용 체크리스트와 프로젝트 트래커 제공

## 주요 문서

### 1. 답변 품질 체크리스트

- `docs/answer_quality_checklist_final.md`
- 기준 데이터: `data/quality_checklist_raw_answer.csv`
- 답변 생성 품질 개선을 위한 최종 체크리스트와 개선활동 정리

### 2. 요약 품질 체크리스트

- `docs/summary_quality_checklist_final.md`
- 기준 데이터: `data/quality_checklist_raw_summary.csv`
- 요약 품질 개선을 위한 최종 체크리스트와 개선활동 정리

### 3. AI RAG 서비스 체크리스트

- `docs/ai_rag_service_checklist.md`
- 범용 AI + RAG 서비스에 공통 적용 가능한 점검 항목 정리
- 데이터, 검색, 컨텍스트, 프롬프트, 모델, 평가, 보안, 운영, 출시 전 점검 포함

### 4. 프로젝트 트래커

- `docs/ai_rag_service_project_tracker.md`
- 체크리스트 항목을 우선순위, 상태, 진행률 기준으로 관리할 수 있는 실행형 문서

## 실무 적용을 위한 환경과 도구

### 권장 운영 환경
- AI 기능이 실제로 연결된 개발·스테이징·운영 환경을 구분해 관리합니다.
- 프롬프트, 모델 설정, 검색 인덱스, 정책 문서를 버전 관리 시스템으로 함께 추적합니다.
- 체크리스트 검토 결과를 이슈 트래커나 협업 도구와 연결해 담당자와 마감일을 명확히 관리합니다.
- 개인정보, 사내 문서, 고객 데이터가 포함되는 경우 접근 권한과 로그 보관 정책을 먼저 확정합니다.

### 필수 입력 자료
- 서비스 요구사항 문서: 사용자 시나리오, 성공 기준, 금지 응답 범위를 정의합니다.
- 평가 데이터셋: 답변형, 요약형, 검색형 시나리오별 대표 질의와 기대 결과를 준비합니다.
- 정책 기준서: 보안, 개인정보, 저작권, 금칙어, 도메인 규정을 정리합니다.
- 운영 기준 정보: SLA, 장애 대응 절차, 배포 승인 절차, 품질 목표 지표를 포함합니다.

### 권장 도구
- 이슈 관리 도구: Jira, Azure DevOps, GitHub Issues 등으로 개선 과제를 추적합니다.
- 문서 협업 도구: Confluence, Notion, Google Docs 등으로 기준 문서와 리뷰 기록을 관리합니다.
- 데이터 관리 도구: CSV, Excel, SQL, BI 도구를 이용해 평가 결과를 누적 관리합니다.
- 실험 및 프롬프트 관리 도구: 프롬프트 버전, 모델 버전, 파라미터 변경 이력을 남깁니다.
- 로그 및 모니터링 도구: 응답 품질, 검색 정확도, 실패율, 지연 시간을 운영 관점에서 관찰합니다.
- 보안 점검 도구: 접근 제어, 비식별 처리, 비밀정보 노출 여부를 확인할 수 있어야 합니다.

### 체크리스트 적용 방식
1. 서비스 유형을 답변형, 요약형, RAG형 중 하나 또는 복합 유형으로 분류합니다.
2. 해당 문서(`docs/*.md`)에서 적용할 점검 항목을 선정하고 프로젝트 기준으로 재정의합니다.
3. 점검 항목별 소유자, 측정 방법, 증빙 자료, 완료 기준을 `docs/ai_rag_service_project_tracker.md`에 기록합니다.
4. 파일럿 평가를 수행해 현재 수준을 확인하고 우선순위가 높은 결함부터 개선합니다.
5. 배포 전 점검, 운영 중 정기 점검, 장애 후 회고에 동일 체크리스트를 반복 적용합니다.

## 권장 작업 순서

1. `data/`의 원본 데이터를 확인하고 현재 프로젝트에 맞는 평가 항목을 선별합니다.
2. `docs/`의 최종 체크리스트 문서를 검토해 적용 범위와 제외 범위를 정의합니다.
3. 실무 환경, 입력 자료, 운영 도구를 준비한 뒤 프로젝트 기준의 품질 목표를 확정합니다.
4. `docs/ai_rag_service_project_tracker.md`에 담당자, 일정, 우선순위, 상태를 기록합니다.
5. 파일럿 점검 결과를 반영해 체크리스트를 보정하고 운영 프로세스에 편입합니다.

## 참고

- `docs/*.md`와 `docs/*.csv`는 체크리스트와 표 형식 결과물을 함께 제공합니다.
- `docs/backup/`에는 이전 버전 또는 중간 산출물이 보관되어 있습니다.
