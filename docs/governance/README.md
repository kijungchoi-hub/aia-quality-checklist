# 개선활동 기준 문서 안내

이 폴더는 `answer` 품질 개선 프로젝트의 개선활동 기준서와 관리 문서를 보관합니다.

## 포함 문서

| 파일명 | 용도 |
|---|---|
| `quality_operating_framework.md` | 전체 개선활동 프레임워크 |
| `search_failure_codebook.md` | 검색 실패 분류 기준 및 초안 사례 |
| `generation_failure_casebook.md` | 생성 실패 사례 초안 |
| `manual_auto_review_log.md` | 수동/자동 평가 통합 로그 초안 |
| `release_gate_checklist.md` | 개선 반영 전 점검표 |
| `weekly_quality_log_2026-W14.md` | 현재 프로젝트 기준 주간 점검 기록 초안 |
| `expansion_validation_matrix.md` | 타 영역 확장 검증 초안 |

## 개선활동 원칙

- 평가 기준, 실패 분류, 회귀 통제 문서는 항상 함께 갱신합니다.
- 문서명은 유지하고 버전이나 주차 정보는 본문 메타데이터로 관리합니다.
- 배포 전에는 최소 `평가셋`, `실패 로그`, `회귀셋`, `배포 게이트` 4종 문서를 함께 점검하는 것을 권장합니다.
