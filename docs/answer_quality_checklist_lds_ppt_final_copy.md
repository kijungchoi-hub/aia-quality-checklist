### 본문 문구

- 수동 검수와 자동 평가를 함께 기록해야 평가 신뢰도가 올라감

### 표 제목

- 검색과 생성 분리 관리 필요성

### 차트 제목

- Retrieval to Generation 파이프라인

### 본문 문구

- 검색 품질은 정답 문서를 회수하는 문제이고 생성 품질은 근거 기반 답변을 만드는 문제임
- Hybrid Search, BM25, Reranker, Top-K 조정이 검색 성능에 직접 영향함
- 검색 성공 이후에도 hallucination, 근거 누락, 답변 왜곡은 별도로 발생함
- 검색 실패와 생성 실패를 분리 측정해야 개선 원인을 정확히 해석할 수 있음

## 별첨 1. 사례 중심 실행 체크리스트

### 목적

- 발표 이후 실제 프로젝트에서 바로 사용할 수 있는 실행형 체크리스트 제공

### 사용 방식

- 각 항목은 `사례`, `점검 질문`, `산출물`, `완료 기준`으로 관리
- 주간 회의에서는 상태를 `미착수 / 진행중 / 완료 / 보류`로 갱신

### 실행 체크리스트

| 영역 | 사례 | 점검 질문 | 산출물 | 완료 기준 |
|---|---|---|---|---|
| 데이터 | 운영 로그 기반 평가셋 재구성 | 최근 질의 패턴이 반영됐는가 | 평가셋 CSV, 질의 유형표 | 예측질의/유도질의/멀티턴 포함 |
| 평가 | 수동+자동 평가 통합 기록 | 정량과 정성 결과가 같은 샘플에 묶여 있는가 | 통합 검수 로그 | 자동 점수와 검수 코멘트 병기 |
| 검색 | 검색 실패 코드북 운영 | 실패가 데이터/색인/메타/쿼리로 분류되는가 | 실패 코드표, 분포표 | 원인별 집계 가능 |
| 생성 | 검색 성공 후 생성 실패 분리 | 정답 Chunk 진입 후 오답이 따로 관리되는가 | 생성 실패 케이스북 | 생성 실패 유형 태깅 완료 |
| 근거 | 답변-출처 매핑 검증 | 답변마다 근거 문서와 Chunk ID가 있는가 | 출처 포맷 정의서 | 근거 없는 답변 fallback 존재 |
| 모델 | 검색/모델 분리 실험 | 검색과 모델 변경 효과가 분리되는가 | 실험 설계표, 기여도 표 | 변수 분리 비교 가능 |
| 운영 | 회귀 테스트셋 운영 | 변경 전후 동일 샘플 비교가 가능한가 | 회귀 테스트셋, 배포 점검표 | 배포 전 회귀 확인 완료 |
| 확산 | 타 영역 교차 검증 | 약관 외 다른 영역에도 효과가 있는가 | 확장 검증표 | 적용 가능성 등급화 완료 |
| 관리 | 주간 포맷 통일 | 수행/결과/회귀/차주 계획이 같은 형식인가 | 주간 템플릿 | 모든 주차 기록 형식 동일 |

### 발표용 요약 문구

- 미착수 항목은 개념이 아니라 실행 단위로 쪼개서 관리해야 함
- 각 항목은 산출물과 완료 기준까지 함께 정의해야 실제 과제로 작동함

## 별첨 2. 실제 프로젝트 운영 프레임워크

### 프레임워크 이름

- Answer Quality Improvement Framework

### 프레임워크 구조

```text
1. Data Readiness
   - 평가셋 최신화
   - 골든셋/정답지/Chunk ID 정합성

2. Retrieval Quality
   - 검색 조합 비교
   - Top-K / Weight / Reranker 최적화
   - 실패 원인 코드화

3. Generation Quality
   - Prompt 개선
   - Hallucination 억제
   - 근거/출처 출력

4. Release Control
   - 회귀 테스트
   - 부작용 점검
   - 배포 게이트

5. Learning Loop
   - 주간 기록 표준화
   - 실패 케이스 축적
   - 타 영역 확산 검토
```

### 운영 원칙

1. 검색 실패와 생성 실패는 항상 분리 관리한다.
2. 모델 변경 전에는 데이터셋과 검색 기준을 먼저 고정한다.
3. 모든 개선은 동일 평가셋으로 전후 비교한다.
4. 배포 전에는 회귀 테스트 결과가 있어야 한다.
5. 주간 기록은 결과뿐 아니라 실패와 차주 계획까지 포함한다.

### 단계별 산출물

| 단계 | 핵심 산출물 |
|---|---|
| Data Readiness | 평가셋, 정합성 점검표 |
| Retrieval Quality | 검색 조합 비교표, 실패 코드북 |
| Generation Quality | 프롬프트 버전표, 출처 포맷 정의서 |
| Release Control | 회귀 테스트셋, 배포 점검표 |
| Learning Loop | 주간 보고서, 실패 케이스북, 확산 검증표 |

## 별첨 3. 실무형 관리 도구 템플릿

### 3-1. 검색 실패 코드북 템플릿

```text
[검색 실패 코드]
DATA  : 정답 문서 자체 부재 또는 잘못된 정답 매핑
INDEX : 문서/청크 색인 누락
META  : 메타데이터 오류로 잘못된 문서 우선 노출
QUERY : 사용자 질의 해석 실패 또는 확장 부족
RANK  : 정답 문서가 있으나 랭킹에서 밀림
```

### 3-2. 생성 실패 케이스북 템플릿

| 샘플 ID | 정답 Chunk 진입 여부 | 생성 결과 | 실패 유형 | 개선 액션 |
|---|---|---|---|---|
| EX-001 | Y | 조건 일부 누락 | 조건 누락 | Prompt 제약 추가 |
| EX-002 | Y | 근거와 다른 표현 | 근거 왜곡 | 출처 인용 규칙 강화 |
| EX-003 | Y | 근거 없는 일반화 | Hallucination | fallback 규칙 적용 |

### 3-3. 회귀 테스트 관리표 템플릿

| 테스트 ID | 카테고리 | 변경 전 | 변경 후 | 회귀 여부 | 비고 |
|---|---|---|---|---|---|
| RG-001 | 약관 | 정답 | 오답 | Y | 배포 보류 |
| RG-002 | FAQ | 정답 | 정답 | N | 유지 |
| RG-003 | 멀티턴 | 부분정답 | 정답 | N | 개선 |

### 3-4. 주간 운영 템플릿

```text
[주간 품질 운영 보고]
1. 이번 주 수행 내용
2. 주요 정량 결과
3. 주요 정성 관찰
4. 실패/회귀 케이스
5. 원인 분석
6. 차주 실행 과제
7. 요청 사항
```

### 3-5. 배포 게이트 체크리스트

| 점검 항목 | 확인 기준 |
|---|---|
| 평가셋 최신성 | 최근 운영 질의 반영 여부 |
| 검색 품질 | 정답 Chunk 진입률 기준 충족 |
| 생성 품질 | hallucination 주요 케이스 통제 |
| 출처 품질 | 답변-근거 매핑 가능 |
| 회귀 통제 | 대표 시나리오 회귀 없음 |
| 운영성 | 기록/모니터링/롤백 기준 존재 |

### 운영 포인트

- 배포 전에는 `평가셋 + 회귀셋 + 배포게이트` 3종 세트가 반드시 준비되어야 함

## 별첨 5. 프로젝트 적용 순서 제안

### 1주차

- 평가셋 최신화
- 검색 실패 코드북 초안 작성
- 주간 운영 템플릿 배포

### 2주차

- 생성 실패 케이스북 구축
- 출처 포함 답변 포맷 적용
