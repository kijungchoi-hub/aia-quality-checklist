# 기록 문서 안내

이 폴더는 PM과 프로젝트 리더가 AI 품질 운영 과정에서 사용하는 기록 문서와 기록 템플릿을 관리하는 영역입니다.

현재 `records` 폴더는 개별 결과 문서 저장소라기보다, 실험, 검증, 회귀, 주간 점검 기록을 남기기 위한 템플릿 중심 구조로 운영합니다.

## 폴더 구성

- [`templates`](templates/): 품질 운영 기록에 사용하는 공통 템플릿 모음

## 권장 사용 방식

1. 실험 설계, 실패 사례 정리, 회귀 테스트, 주간 점검이 필요하면 [`templates`](templates/)에서 해당 템플릿을 선택합니다.
2. 작성된 결과 문서는 프로젝트 운영 방식에 따라 `records` 하위에 추가하거나 별도 운영 저장 위치에서 관리합니다.
3. 품질 판단 기준과 배포 기준은 [`../governance`](../governance/)와 연결해서 사용합니다.

## 사용 원칙

- `records`는 결과 기록과 운영 이력 관리 목적의 영역입니다.
- 공통 기준 문서는 [`../checklists`](../checklists/)와 [`../governance`](../governance/)에서 확인합니다.
- 실행 계획과 담당/상태 관리는 [`../projects`](../projects/)에서 관리합니다.
- 템플릿 설명은 [`templates/README.md`](templates/README.md)를 기준으로 확인합니다.
