# 답변 품질 체크리스트

기준 데이터: [`quality_checklist_raw_answer.csv`](../../data/quality_checklist_raw_answer.csv)

정리 기준:
- 문서 유형별 구분은 제거합니다.
- 주차별 반복 표현은 제거합니다.
- 동일한 목적의 개선 활동은 하나로 통합합니다.
- 실행 가능한 체크리스트와 개선 활동만 남깁니다.

## 최종 체크리스트

### 1. 데이터 및 평가 기준

- [ ] 학습 및 검증 세트를 확보하고 최신 상태로 유지합니다.
- [ ] 골든셋, 정답지, Chunk ID 간 정합성을 검증합니다.
- [ ] Baseline 정확도와 실패 사례를 재검합니다.
- [ ] 평가 지표를 정답률, 오답률, 소답률, 과답률, 검색 매칭률로 통일합니다.
- [ ] 튜닝 전후 비교 테스트 기준을 동일하게 유지합니다.
- [ ] 수동 검수 결과와 자동 평가 결과를 함께 기록합니다.

### 2. 검색 품질 개선

- [ ] Hybrid Search 성능을 검증하고 기본 검색 방식으로 확정합니다.
- [ ] Lexical, Vector, BM25, Reranker 조합별 성능을 비교합니다.
- [ ] Top-K, 리랭킹, Weight 조합을 최적화합니다.
- [ ] Query Expansion 적용 여부와 효과를 검증합니다.
- [ ] 검색 노이즈와 정답 Chunk의 랭킹 진입 여부를 점검합니다.
- [ ] 검색 실패 원인을 데이터, 색인, 메타, 쿼리 단계로 분리합니다.

### 3. 프롬프트 및 생성 품질 개선

- [ ] 답변 생성 프롬프트 개선 포인트를 정리합니다.
- [ ] Hallucination 절감용 제약 조건을 적용하고 효과를 검증합니다.
- [ ] 프롬프트 축소, 단순화, 구조화 실험 결과를 비교합니다.
- [ ] Few-shot, Anchoring, Soft routing 적용 여부를 검토합니다.
- [ ] 검색 성공 후 생성 실패 케이스를 별도 관리합니다.
- [ ] 답변 근거와 출처를 함께 생성하는 흐름을 검증합니다.

### 4. 컨텍스트 및 워크플로우 개선

- [ ] Context Refinement 적용 여부를 결정합니다.
- [ ] 상위 1~3개 컨텍스트만 사용하는 방식의 성능을 검증합니다.
- [ ] Self-Verification, Self-Consistency 등 후처리 전략의 실효성을 비교합니다.
- [ ] Workflow 분리 또는 분류형 접근이 필요한 영역을 식별합니다.
- [ ] 문서 간 오염과 영향도를 점검합니다.

### 5. 모델 튜닝 및 개선 적용

- [ ] SFT 기반 기본 모델을 확정합니다.
- [ ] DPO, KTO, SimPO, IPO, ORPO/Hinge 등 선호도 학습 전략을 비교합니다.
- [ ] 최적 튜닝 모델의 전수 평가 결과를 기록합니다.
- [ ] 모델 튜닝 효과와 검색 개선 효과를 분리해서 해석합니다.
- [ ] 개선 반영 전 회귀 케이스와 부작용을 점검합니다.
- [ ] 타 영역 확장 적용 가능 여부를 판단합니다.

### 6. 오류 분석 및 개선 관리

- [ ] 실패 케이스를 검색 실패와 답변 생성 오류로 분류합니다.
- [ ] 미매칭 케이스는 문서 미검색과 문서 매칭 오류로 나누어 관리합니다.
- [ ] Reference 오기입, 색인 누락, 청크 오매핑 등 데이터 품질 이슈를 수정합니다.
- [ ] 외부 협의가 필요한 항목은 별도 이슈로 추적합니다.
- [ ] 주차별 수행, 결과, 회귀, 차주 계획을 같은 포맷으로 기록합니다.

## 최종 개선활동

### 데이터 정비

- 학습 및 검증 세트 수급
- 골든셋 기준 정답 데이터와 Chunk ID 매칭
- 정답지와 실데이터 간 불일치 식별
- 수동 검수 및 데이터 품질 이슈 정리

### 검색 개선

- Hybrid Search 적용
- BM25 및 형태소 분석 기반 검색 보강
- Reranker 교체 및 Top-K 조정
- Query Expansion 적용
- 정답 Chunk 랭킹 진입률 점검
- 검색 노이즈 제거 및 Weight 재산정

### 프롬프트 및 생성 개선

- 답변 생성 프롬프트 개선
- Hallucination 억제 조건 추가
- Prompt 단순화 및 축소 실험
- Few-shot, Anchoring, 질의 유형 분류 적용
- 검색 성공 후 답변 실패 케이스 집중 개선

### 컨텍스트 최적화

- Context Refinement 적용
- 상위 컨텍스트 필터링 실험
- Self-Verification 적용
- Self-Consistency 적용 여부 검토
- 문서 간 영향도 및 오염 분석

### 모델 튜닝

- SFT 학습 데이터 설계 및 학습
- DPO, KTO, SimPO, IPO, ORPO/Hinge 실험
- 튜닝 모델 A/B 비교 평가
- 전수 테스트 기반 회귀 및 교정 분석
- 개선 반영 전 안정성 검토

### 개선활동 및 추적

- 답변 근거와 출처 생성 흐름 점검
- 외부 협의 필요 항목 추적
- 주차별 실험 로그 표준화
- KPI와 카테고리별 결과 기록 체계 정리

## 제외된 항목

- 특정 문서 유형 이름
- 동일한 의미를 반복하는 주차별 표현
- 개별 카테고리에만 종속된 임시 메모
