# AI RAG 서비스 체크리스트 표

| 구분 | 세부 항목 | 우선순위 | 상태 | 비고 |
|---|---|---|---|---|
| 데이터 및 지식베이스 준비 | 수집 대상 문서와 데이터 소스를 정의 | 높음 | 진행중 | 내부/외부 소스 구분 포함 |
| 데이터 및 지식베이스 준비 | 최신성, 신뢰도, 접근권한 기준으로 데이터 소스를 분류 | 높음 | 진행중 | 권한 정책과 연계 |
| 데이터 및 지식베이스 준비 | 원문, 요약, 메타데이터, 출처 정보를 함께 저장 | 높음 | 진행중 | 검색과 근거 제시에 필요 |
| 데이터 및 지식베이스 준비 | 문서 청크 기준을 정의하고 일관되게 적용 | 높음 | 진행중 | chunk size/overlap 기준 포함 |
| 데이터 및 지식베이스 준비 | 중복 문서, 오래된 문서, 충돌 문서를 정리 | 중간 | 미착수 | 버전 정책 필요 |
| 데이터 및 지식베이스 준비 | 색인 누락, 메타 누락, 청크 오매핑 이슈를 추적 | 높음 | 진행중 | 데이터 품질 이슈 관리 |
| 데이터 및 지식베이스 준비 | 버전 변경 시 재색인 정책을 정의 | 중간 | 미착수 | 재색인 트리거 조건 정의 |
| 검색 품질 | 기본 검색 전략을 정의 | 높음 | 진행중 | Lexical/Vector/Hybrid 비교 전제 |
| 검색 품질 | Lexical, Vector, Hybrid 검색을 비교 | 높음 | 진행중 | 정답 문서 회수율 비교 |
| 검색 품질 | BM25, Dense Retrieval, Reranker 조합을 검증 | 높음 | 진행중 | 검색 조합별 평가 필요 |
| 검색 품질 | Top-K, Score Threshold, Weight 조합을 최적화 | 높음 | 진행중 | 정확도/비용/지연시간 균형 |
| 검색 품질 | Query Expansion 또는 Query Rewrite 적용 여부를 검토 | 중간 | 진행중 | 도메인별 효과 차이 확인 |
| 검색 품질 | 정답 문서가 상위 랭킹에 진입하는지 점검 | 높음 | 진행중 | retrieval recall 핵심 |
| 검색 품질 | 검색 실패 원인을 색인, 메타, 쿼리, 랭킹 단계로 분리 | 높음 | 미착수 | 원인 코드 체계 필요 |
| 검색 품질 | 검색 지연시간과 비용을 함께 측정 | 중간 | 미착수 | 운영성 관점 필요 |
| 컨텍스트 구성 | 검색 결과를 그대로 넣을지, 정제된 컨텍스트만 넣을지 결정 | 높음 | 진행중 | 정확도와 단순성 비교 |
| 컨텍스트 구성 | Context Refinement(컨텍스트 정제) 적용 여부를 검토 | 높음 | 진행중 | 정확도/지연시간 비교 |
| 컨텍스트 구성 | 상위 1~N개 컨텍스트만 사용하는 방식의 성능을 비교 | 높음 | 진행중 | 회귀 여부 함께 확인 |
| 컨텍스트 구성 | 중복 청크, 노이즈 청크, 충돌 청크 제거 규칙을 정의 | 중간 | 미착수 | 전처리 규칙 필요 |
| 컨텍스트 구성 | 서로 다른 문서 간 내용 충돌 시 우선순위 정책을 정함 | 중간 | 미착수 | 출처 우선순위 필요 |
| 컨텍스트 구성 | 근거 문서와 답변 문맥의 연결 가능성을 검증 | 높음 | 진행중 | grounding 검증 |
| 프롬프트 및 생성 품질 | 시스템 프롬프트와 태스크 프롬프트의 역할을 분리 | 높음 | 진행중 | 책임 분리 명확화 |
| 프롬프트 및 생성 품질 | 답변 생성 프롬프트의 목표, 제약, 출력 형식을 정의 | 높음 | 진행중 | 일관된 출력 보장 |
| 프롬프트 및 생성 품질 | Hallucination(환각) 억제 규칙을 추가 | 높음 | 진행중 | 정보 부족 시 정책 포함 |
| 프롬프트 및 생성 품질 | 정보 부족 시 안전하게 거절하거나 보류하는 문구를 정의 | 높음 | 미착수 | 안전 정책과 연계 |
| 프롬프트 및 생성 품질 | Few-shot(소수 예시 학습), 규칙, Anchoring(기준 고정), Routing(질문 분기) 적용 여부를 검토 | 중간 | 진행중 | 태스크별 차등 적용 |
| 프롬프트 및 생성 품질 | Prompt 축소, 단순화, 구조화 실험 결과를 비교 | 중간 | 진행중 | 응답속도와 품질 비교 |
| 프롬프트 및 생성 품질 | 검색 성공 후 생성 실패 케이스를 별도 관리 | 높음 | 미착수 | 생성 문제 분리 필요 |
| 프롬프트 및 생성 품질 | 답변과 함께 근거, 출처, 신뢰 수준 제공 여부를 결정 | 중간 | 미착수 | UX와 신뢰성 연계 |
| 모델 선택 및 튜닝 | 기본 생성 모델 후보를 비교 | 높음 | 진행중 | 성능/비용/지연시간 포함 |
| 모델 선택 및 튜닝 | 성능, 비용, 지연시간, 개선 적용 제약 조건을 함께 평가 | 높음 | 진행중 | 개선 적합성 판단 |
| 모델 선택 및 튜닝 | SFT 적용 여부와 학습 데이터 기준을 정의 | 중간 | 미착수 | 도메인 적합성 확인 |
| 모델 선택 및 튜닝 | 선호도 학습 전략을 비교 | 중간 | 미착수 | DPO/KTO/SimPO/IPO/ORPO 등 |
| 모델 선택 및 튜닝 | 튜닝 효과와 검색 개선 효과를 분리해서 해석 | 높음 | 미착수 | 실험 설계 필요 |
| 모델 선택 및 튜닝 | 튜닝 모델의 회귀 케이스를 별도로 추적 | 높음 | 미착수 | 개선 반영 전 필수 |
