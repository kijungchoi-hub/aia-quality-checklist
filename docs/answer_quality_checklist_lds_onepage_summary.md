# Answer Checklist 기반 LDS 1페이지 요약

기준:

## 개요

`answer` 품질 체크리스트를 기준으로 답변 품질 개선 활동을 LDS 관점에서 재정리했다. 핵심은 답변 품질을 모델 단일 문제로 보지 않고, 데이터 정합성, 검색 회수율, 생성 안정성, 운영 추적 체계의 결합 문제로 해석하는 것이다.

## 현황

| 지표 | 값 |
|---|---:|
| 전체 항목 | 34 |
| 진행중 | 25 |
| 미착수 | 9 |
| 높음 우선순위 | 24 |
| 높음 + 미착수 | 5 |

## Learn

- 평가 신뢰도는 골든셋, 정답지, Chunk ID 정합성 확보에 달려 있다.
- 검색 성공과 생성 성공은 별개이므로 분리 측정이 필요하다.
- Context Refinement와 상위 컨텍스트 선별이 노이즈 감소에 효과적이다.
- 모델 튜닝은 검색 개선 효과와 분리해서 해석해야 한다.
- 실패 유형 분류와 회귀 추적이 운영 안정화의 핵심이다.

## Do

- 학습 및 검증 세트를 확보하고 최신화한다.
- Hybrid Search, BM25, Reranker, Top-K 조합을 최적화한다.
- hallucination 억제 규칙과 출처 생성 흐름을 강화한다.
- 검색 성공 후 생성 실패 케이스를 별도 관리한다.
- 운영 반영 전 회귀 케이스와 부작용을 점검한다.

## Share

- 답변 품질 개선은 데이터, 검색, 생성, 운영이 함께 맞물린 구조라는 점을 공유한다.
- 평가셋 정합성과 정답 Chunk 매핑 기준을 공통 자산으로 정리한다.
- 검색 조합 비교 기준과 생성 실패 분리 관리 기준을 타 팀에 확산한다.
- 주차별 동일 포맷 기록 체계를 조직 표준으로 제안한다.

## 한 줄 결론

답변 품질 개선의 핵심은 모델 교체가 아니라, 신뢰 가능한 평가 기반 위에서 검색과 생성을 분리 개선하고 이를 운영 가능한 형태로 축적하는 것이다.
