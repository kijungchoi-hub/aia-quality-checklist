## 슬라이드 1. 제목

### 제목

- Answer Quality Checklist 기반 LDS 보고

### 부제

- 데이터, 검색, 생성, 운영 관점의 답변 품질 개선 정리

### 장표 문구

- 대상: `answer` 품질 체크리스트
- 관점: Learn / Do / Share
- 작성일: 2026-04-01

### 도식 초안 텍스트

```text
[Data] [Search] [Generation] [Operations]
        \    Learn / Do / Share    /
```

---

## 슬라이드 2. 추진 배경

### 제목

- 왜 체크리스트 기반 재정리가 필요했는가

### 장표 문구

- 개선 과제가 영역별로 분산 관리됨
- 동일 문제의 중복 관리 가능성 존재
- 운영 반영 기준과 회귀 기준이 불명확
- 실행 가능한 공통 프레임 필요

### 도식 초안 텍스트

```text
Before: 실험별 메모 / 산발적 개선 / 결과 해석 분산
   ->
After: 체크리스트 통합 / LDS 구조화 / 운영 연결 가능
```

## 슬라이드 3. 전체 현황

### 제목

- 체크리스트 전체 현황

### 장표 문구

- 전체 항목: 34
- 진행중: 25
- 미착수: 9
- 높음 우선순위: 24
- 높음 우선순위 미착수: 5

### 한 줄 메시지

- 핵심 축은 착수됐고, 남은 공백은 우선순위 기준으로 메워야 함

### 도표 초안 텍스트

```text
[상태]
진행중 25  #########################
미착수  9  #########

[우선순위]
높음 24  ########################
중간 10  ##########
```

### 표 초안 텍스트

```text
구분                  건수

미착수                 9

```

## 슬라이드 4. Learn 1

### 제목

- 데이터와 평가 기준이 결과 해석의 출발점

### 장표 문구

- 골든셋, 정답지, Chunk ID 정합성 확보 필요
- Baseline 정확도와 실패 사례 재검 필요
- 평가 지표 통일 필요
- 수동 검수와 자동 평가 병행 필요

### 한 줄 메시지

- 평가 기반이 흔들리면 개선 효과를 신뢰할 수 없음

### 도식 초안 텍스트

```text
          [실험 결과 해석]
        [평가 기준 통일]
      [데이터 정합성 확보]
```

---

## 슬라이드 5. Learn 2

### 제목

- 검색 성공과 생성 성공은 다른 문제

### 장표 문구

- 검색은 정답 문서 회수 문제
- 생성은 근거 기반 답변 생성 문제
- 검색 성공 후에도 생성 실패 발생
- 검색과 생성을 분리 측정해야 함

### 한 줄 메시지

- 검색과 생성이 섞이면 원인 분석이 왜곡됨

### 도식 초안 텍스트

```text
Query
  ->
Retrieval
  ->  [실패 A: 문서 미검색 / 오매칭]
Context
  ->
Generation
  ->  [실패 B: hallucination / 근거 누락 / 왜곡]
Answer
```

## 슬라이드 6. Learn 3

### 제목

- 많이 넣는 것보다 정확하게 넣는 것이 중요

### 장표 문구

- Context Refinement 검토 필요
- 상위 1~3개 컨텍스트 선별 실험 필요
- Self-Verification 비교 필요
- 문서 간 오염도 점검 필요

### 한 줄 메시지

- 컨텍스트 정제와 워크플로우 설계가 품질 차이를 만듦

### 도식 초안 텍스트

```text
Case A: 다수 컨텍스트 투입
- 정보량 많음
- 노이즈 높음
- 품질 변동 큼

Case B: 상위 컨텍스트 정제
- 정보량 적정
- 노이즈 낮음
- 품질 안정화
```

## 슬라이드 7. Do 1

### 제목

- 단기 최우선 실행 과제

### 장표 문구

- 학습 및 검증 세트 확보
- 검색 실패 원인 코드 체계화
- 생성 실패 케이스 분리 관리
- 튜닝 효과와 검색 효과 분리 해석
- 운영 반영 전 회귀 점검

### 한 줄 메시지

- 신규 실험 확대보다 현재 공백 해소가 우선

### 표 초안 텍스트

```text
과제                         목적
학습/검증 세트 확보          평가 신뢰도 확보
실패 원인 코드화            분석 정밀도 향상
생성 실패 분리 관리         원인 분리
효과 분리 해석              오판 방지
회귀 점검                   운영 안정성 확보
```

---

## 슬라이드 8. Do 2

### 제목

- 실행은 네 축으로 정리

### 장표 문구

- 데이터: 정합성, Baseline, 평가 로그
- 검색: Hybrid, BM25, Reranker, Top-K
- 생성: Prompt, Hallucination 억제, 출처 생성
- 운영: 실패 분류, 회귀 추적, 기록 표준화

### 한 줄 메시지

- 데이터, 검색, 생성, 운영이 함께 정렬돼야 품질이 안정화됨

### 도식 초안 텍스트

```text
┌ 데이터 ─ 정합성 / 기준 / 평가로그
├ 검색   ─ 회수율 / 랭킹 / 조합 최적화
├ 생성   ─ 프롬프트 / 근거 / 안정성
└ 운영   ─ 실패분류 / 회귀 / 기록표준
```

## 슬라이드 9. Share

### 제목

- 이번 정리의 가치는 공통 기준을 만든 데 있음

### 장표 문구

- 평가셋 정합성 기준 공유
- 검색/생성 분리 측정 기준 공유
- 실패 유형 분류 체계 공유
- 주차별 기록 포맷 표준화 제안

### 한 줄 메시지

- 이번 결과물은 현재 과제를 넘어 조직 공통 자산으로 확장 가능

### 도식 초안 텍스트

```text
           [Answer Checklist Framework]
        /        |         |         |        \
    Data     Search    Prompt    Model      Ops
```

## 슬라이드 10. 결론

### 제목

- 구조를 정렬해야 품질이 올라간다

### 장표 문구

- 핵심은 모델 교체보다 구조적 개선
- 우선순위 미착수 5건부터 착수
- 실패 분류와 회귀 기준 문서화
- 운영 가능한 개선 체계로 전환

### 한 줄 메시지

- 답변 품질 개선은 구조적 정렬의 문제

### 도식 초안 텍스트

```text
평가 신뢰도 확보
   ->
검색/생성 분리 개선
   ->
회귀 관리 체계화
   ->
운영 가능한 품질 개선
```

## 발표자 메모용 초단문

- Learn: 무엇을 배웠는가
- Do: 무엇을 우선 실행할 것인가
- Share: 무엇을 공통 자산으로 만들 것인가

- 핵심 포인트 1: 평가 기반 신뢰도
- 핵심 포인트 2: 검색/생성 분리
- 핵심 포인트 3: 운영 추적 체계
