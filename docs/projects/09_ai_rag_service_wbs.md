# AI RAG 서비스 품질 운영 WBS 초안

기준 문서:
- [`01_ai_rag_service_checklist.md`](../checklists/01_ai_rag_service_checklist.md)
- [`08_ai_rag_service_project_tracker.md`](08_ai_rag_service_project_tracker.md)
- [`09_quality_operating_framework.md`](../governance/09_quality_operating_framework.md)

이 문서는 PM과 프로젝트 리더가 AI RAG 서비스 품질 운영(개선/관리)을 실행 가능한 작업 단위로 분해해 일정, 담당, 산출물, 완료 기준으로 관리할 수 있도록 정리한 WBS 초안입니다.

## 1. WBS 사용 원칙

- 상위 WBS는 품질 운영의 큰 영역을 뜻합니다.
- 하위 WBS는 실제 일정과 담당자 배정이 가능한 작업 단위입니다.
- 각 작업은 반드시 산출물과 완료 기준을 함께 둡니다.
- 상세 일정과 상태 관리는 [`08_ai_rag_service_project_tracker.md`](08_ai_rag_service_project_tracker.md)에서 이어서 관리합니다.

## 2. 상위 WBS 구조

| WBS | 작업명 | 목적 |
|---|---|---|
| 1 | 프로젝트 착수 및 범위 정의 | 적용 범위, 품질 목표, 운영 조건을 확정 |
| 2 | 데이터 및 지식베이스 준비 | 검색과 답변의 기반 데이터 품질 확보 |
| 3 | 검색 품질 운영 | 정답 문서 회수율과 검색 안정성 확보 |
| 4 | 컨텍스트 구성 운영 | 검색 결과를 생성 입력으로 안정적으로 연결 |
| 5 | 프롬프트 및 생성 품질 운영 | 응답 품질과 생성 안정성 확보 |
| 6 | 모델 선택 및 튜닝 운영 | 모델 적합성, 비용, 성능, 회귀 리스크 관리 |
| 7 | 평가 체계 운영 | 품질 판단 기준과 비교 체계 확립 |
| 8 | 오류 분석 및 개선 루프 운영 | 실패 원인 분해와 반복 개선 체계 구축 |
| 9 | 보안 및 정책 운영 | 권한, 민감정보, 안전 정책 리스크 통제 |
| 10 | 운영 관측성 및 기록 관리 | 로그, 지표, 변경 이력, 주간 기록 축적 |
| 11 | 배포 게이트 및 운영 반영 | 운영 반영 전 최종 판단과 승인 |
