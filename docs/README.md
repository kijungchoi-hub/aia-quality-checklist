# 문서 안내

## 폴더 구성

- `checklists/`: 프로젝트 시작 전에 기준으로 삼는 최종 체크리스트를 둡니다.
- `projects/`: 실행 계획과 진행 상태를 관리합니다.
- `governance/`: 개선활동 기준, 실패 분류, 배포 기준을 관리합니다.
- `records/`: 개선활동 개요 문서와 개선활동 기록 문서를 관리합니다.
- `records/templates/`: 실험, 검증, 배포 점검, 주간 기록 템플릿을 보관합니다.

## 권장 사용 순서

1. `checklists/`에서 적용 기준 문서를 고릅니다.
2. `projects/`에서 실행 계획과 우선순위를 기록합니다.
3. `governance/`에서 개선활동 기준과 검증 기준을 연결합니다.
4. `records/templates/`에서 필요한 템플릿을 골라 산출물을 작성합니다.
5. `records/`에서 개선활동 개요와 주간 기록을 정리합니다.

## 개선활동 원칙

- 최종 기준 문서는 `checklists/`에 둡니다.
- 실행 중 문서는 `projects/`에 둡니다.
- 정책과 게이트 기준은 `governance/`에 둡니다.
- 개선활동 개요와 주간 기록은 `records/`에 둡니다.
- 이전 버전과 중간 산출물은 최상위 `archive/`로 이동해 보관합니다.
