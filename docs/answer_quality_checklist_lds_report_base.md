## 1. 보고서 목적

본 문서는 `answer` 품질 체크리스트를 기준으로 LDS(Learn, Do, Share) 보고서 작성 시 바로 활용할 수 있도록 핵심 학습 내용, 실행 과제, 공유 포인트를 재구성한 기초자료이다.  
체크리스트의 현재 상태를 기반으로 우선 실행이 필요한 항목과 보고서 서술 포인트를 함께 정리한다.

## 3. LDS 초안

## Learn

### 3-1. 이번 활동에서 확인한 핵심 학습

1. 답변 품질은 모델 자체보다 데이터 정합성과 평가 기준 통일의 영향을 크게 받는다.
2. 검색 품질이 확보되지 않으면 프롬프트나 모델 튜닝만으로는 정답률 개선에 한계가 있다.
3. 검색 성공 이후에도 생성 단계에서 hallucination, 근거 누락, 답변 왜곡이 별도로 발생한다.
4. 상위 컨텍스트 선별, Context Refinement, Self-Verification 같은 워크플로우 요소가 답변 품질에 유의미한 영향을 준다.
5. 모델 튜닝 효과와 검색 개선 효과를 분리해서 해석하지 않으면 개선 원인을 잘못 판단할 위험이 있다.
6. 운영 단계에서는 실패 유형 분류, 회귀 추적, 주차별 기록 표준화가 품질 유지에 필수다.

### 3-2. 체크리스트 기반 세부 학습 포인트

| 학습 주제 | 핵심 인사이트 | 보고서에 넣을 표현 예시 |
|---|---|---|
| 데이터/평가 | 골든셋, 정답지, Chunk ID 정합성이 흔들리면 평가 결과 신뢰도가 낮아짐 | 답변 품질 개선 이전에 평가 기반의 신뢰성 확보가 선행되어야 함 |
| 검색 | Hybrid Search, BM25, Reranker, Top-K 조합이 정답 Chunk 진입률에 직접 영향 | 검색 조합 최적화가 정답률 향상의 선행 조건으로 확인됨 |
| 생성 | Prompt 구조화와 hallucination 억제 조건이 생성 품질 안정화에 필요 | 검색 성공 후에도 생성 실패를 별도 관리해야 함 |
| 컨텍스트 | 상위 컨텍스트 제한과 refinement가 노이즈를 줄임 | 많이 넣는 것보다 정확한 컨텍스트 선별이 중요함 |
| 모델 튜닝 | SFT 및 선호도 학습 전략은 비교 실험과 전수평가가 필요 | 튜닝 자체보다 운영 적합성과 회귀 위험을 함께 봐야 함 |
| 운영 관리 | 실패 분류 체계와 주차별 동일 포맷 기록이 개선 속도를 높임 | 개선 활동을 재현 가능한 형태로 남기는 체계가 중요함 |

## Do

### 4-1. 즉시 실행 우선 과제

아래 항목은 체크리스트에서 `높음 + 미착수` 또는 보고서상 즉시 액션으로 강조할 가치가 큰 과제들이다.

| 우선순위 | 실행 과제 | 기대 효과 |
|---|---|---|
| 1 | 학습 및 검증 세트 확보 및 최신화 | 평가 신뢰도 확보 |
| 2 | 검색 실패 원인 코드 체계화 | 개선 포인트를 데이터/색인/메타/쿼리 단위로 분리 가능 |
| 3 | 검색 성공 후 생성 실패 케이스 분리 관리 | 생성 품질 이슈의 독립 개선 가능 |
| 4 | 모델 튜닝 효과와 검색 개선 효과 분리 해석 | 개선 원인에 대한 오판 방지 |
| 5 | 운영 반영 전 회귀 케이스와 부작용 점검 | 운영 배포 리스크 감소 |

### 4-2. 영역별 실행 과제 정리

| 영역 | 실행 과제 |
|---|---|
| 데이터 및 평가 기준 | 골든셋 정합성 재검, Baseline 재측정, 평가 지표 통일, 수동/자동 평가 동시 기록 |
| 검색 품질 개선 | Hybrid Search 검증, 검색 조합 비교, Top-K/Weight 최적화, Query Expansion 효과 검증 |
| 프롬프트 및 생성 품질 개선 | 프롬프트 개선, hallucination 억제 규칙 적용, 구조화 실험, 출처 생성 흐름 검증 |
| 컨텍스트 및 워크플로우 개선 | Context Refinement, 상위 컨텍스트 필터링, Self-Verification 비교, 분류형 워크플로우 검토 |
| 모델 튜닝 및 운영 적용 | SFT 기반 모델 확정, 선호도 학습 전략 비교, 전수평가, 회귀 분석 |
| 오류 분석 및 운영 관리 | 실패 유형 분류, 미매칭 세분화, 데이터 품질 이슈 수정, 외부 협의 항목 추적 |

### 4-3. 보고서용 실행 문장 초안

- 데이터 정합성과 평가 기준 통일을 우선 과제로 두고, 품질 측정의 신뢰도를 먼저 확보했다.
- 검색 품질 개선은 Hybrid Search, BM25, Reranker, Top-K 조정 중심으로 병행 검증했다.
- 검색 성공 후 생성 실패를 별도 관리하면서 프롬프트 개선과 hallucination 억제 규칙을 보강했다.
- 컨텍스트 정제와 후처리 전략을 비교하여 grounding 안정화를 추진했다.
- 모델 튜닝은 SFT 및 선호도 학습 전략 비교와 전수평가를 통해 운영 반영 가능성을 검토했다.
- 실패 유형 분류와 주차별 기록 표준화를 통해 운영 개선 루프를 구조화했다.

## Share

### 5-1. 조직 내 공유 가치

1. 답변 품질 문제는 단일 모델 문제로 보지 않고 데이터, 검색, 생성, 운영 체계의 결합 문제로 봐야 한다.
2. 검색 정확도와 생성 정확도를 분리 측정해야 개선 효과를 정확히 설명할 수 있다.
3. 골든셋과 Chunk ID 정합성 검증은 모든 후속 실험의 전제 조건이다.
4. hallucination 억제는 프롬프트 문구 추가만으로 끝나지 않고 근거 생성 흐름과 함께 설계되어야 한다.
5. 회귀 테스트와 주차별 동일 포맷 기록은 운영 반영 전에 반드시 필요한 관리 체계다.

### 5-2. 타 팀/타 과제 확산 포인트

| 공유 대상 | 공유 포인트 | 기대 효과 |
|---|---|---|
| 데이터 담당 | 골든셋-정답지-Chunk ID 정합성 관리 기준 | 평가 기반 신뢰도 향상 |
| 검색 담당 | Hybrid Search 및 Reranker 조합 비교 기준 | 검색 실험 재사용성 확보 |
| 프롬프트 담당 | hallucination 억제 규칙과 생성 실패 분리 관리 | 생성 품질 개선 속도 향상 |
| 모델 담당 | 튜닝 효과와 검색 효과 분리 실험 설계 | 튜닝 투자 우선순위 명확화 |
| 운영 담당 | 실패 유형 분류, 회귀 케이스 추적, 주차별 기록 포맷 | 운영 안정성 및 보고 체계 강화 |

### 5-3. 공유 메시지 초안

- 답변 품질 개선은 검색과 생성을 분리해 봐야 정확한 원인 분석이 가능하다.
- 평가셋과 정답 매핑의 정합성이 확보되어야 실험 결과를 신뢰할 수 있다.
- 프롬프트 개선만으로 해결되지 않는 문제는 컨텍스트 설계와 워크플로우 보완이 필요하다.
- 모델 튜닝은 반드시 회귀와 운영 안정성 관점에서 함께 검토해야 한다.
- 주차별 결과, 실패, 회귀, 차주 계획을 같은 포맷으로 남겨야 조직 학습이 축적된다.

## 6. 보고서 작성용 한 페이지 요약

### Learn

- 답변 품질은 데이터 정합성, 검색 성능, 생성 제약, 운영 추적 체계가 함께 맞물려 결정된다.
- 검색 성공과 생성 성공은 별개이며, 두 단계를 분리 분석해야 개선 효율이 높다.
- 모델 튜닝은 독립 효과 검증과 회귀 관리 없이 운영 반영하기 어렵다.

### Do

- 평가 기준 통일, 검색 조합 최적화, 프롬프트 개선, 컨텍스트 정제, 튜닝 모델 전수평가를 병행 추진한다.
- 높음 우선순위 미착수 항목을 먼저 해소해 운영 반영 리스크를 낮춘다.
- 실패 유형 분류와 주차별 기록 표준화를 통해 개선 루프를 고정한다.

### Share

- 데이터 정합성 기준, 검색 실험 기준, 생성 실패 관리 기준, 회귀 추적 포맷을 팀 공통 자산으로 공유한다.
- 타 영역에도 동일한 체크리스트 구조를 확장 적용할 수 있도록 문서화한다.

## 7. 후속 문서화 권장 항목

- LDS 주간 보고서 템플릿 추가 작성
- 실패 케이스 분류 코드 표준안 작성
- 회귀 테스트셋 운영 기준 문서화
- 수동 검수 로그 템플릿 작성
