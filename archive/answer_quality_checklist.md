# Answer Quality Improvement Checklist

기준 데이터: `data/quality_checklist_raw_answer.csv`

집계 범위: `2026-01-09`부터 `2026-03-27`까지 모든 답변 품질개선 이력

이 문서는 주차별 원본을 그대로 복붙하지 않고, 누적된 개선 과제를 하나의 실행 체크리스트로 정리한 문서입니다.

## 공통

### 누적 수행 내용

- Top-30 리랭킹, 통합 Rerank, Top-k 조정 실험 수행
- Hybrid Search, Lexical + Vector 비중 조정, 형태소 분석기 적용 실험 수행
- RAGAS 평가 방식과 Judge 프롬프트 구축
- 정답 데이터 Chunk ID 매칭 및 골든셋 정합성 검증 수행
- 프롬프트 Ablation, Query Expansion, Context Refinement 실험 수행
- SFT, DPO, KTO, SimPO, IPO, ORPO/Hinge 계열 튜닝 실험 진행
- 오답노트 작성과 오류 유형 분류 체계 수립

### 체크리스트

- [ ] 통합 검색 파이프라인의 최종 기준안을 확정합니다.
- [ ] Hybrid Search, Reranker, Query Expansion, Context Refinement 조합별 성능표를 정리합니다.
- [ ] 정답 Chunk ID 자동 수집 및 갱신 모듈을 운영 가능한 수준으로 정리합니다.
- [ ] 프롬프트 개선 효과와 모델 튜닝 효과를 분리해서 비교합니다.
- [ ] ORPO/Hinge 포함 선호도 학습 계열의 최종 채택 알고리즘을 확정합니다.
- [ ] 전 카테고리 공통 KPI를 다시 정의하고 동일 포맷으로 주차별 기록을 남깁니다.
- [ ] 답변 출처와 근거 생성 흐름을 Workflow 기준으로 검증합니다.
- [ ] 오답 유형을 검색 실패, 매칭 실패, 생성 실패로 고정 분류해 추적합니다.

## 업무 매뉴얼

### 누적 수행 내용

- 도메인 확장, 질병/키워드 Anchoring, Few-shot, Soft routing 실험 수행
- 용어집 기반 Query Expansion과 Prompt 축소 실험 수행
- 문서 Rewrite, 약어 사전, 현업 피드백 반영 검토
- Self-Consistency와 Chunk Attention 계열 실험 수행
- CR 파이프라인과 Qwen3-30B 기반 모델 튜닝 착수
- SFT 결과 기반 DPO/SimPO/IPO 비교와 전수평가 수행

### 체크리스트

- [ ] 업무 매뉴얼용 최종 프롬프트 구조를 확정합니다.
- [ ] Context Refinement와 Self-Verification 조합의 실효성을 수치로 검증합니다.
- [ ] 업무 절차 순서와 조건 분기 오류 감소 여부를 별도로 측정합니다.
- [ ] 튜닝 모델 전수평가 결과를 기준으로 베이스라인 대비 개선 폭을 재확인합니다.
- [ ] SimPO, IPO, ORPO/Hinge 중 실제 운영에 넣을 학습 전략을 확정합니다.
- [ ] 도메인 메타 미적재 5개 분류의 검색 보완 방안을 정리합니다.

## PC 매뉴얼

### 누적 수행 내용

- 다른 지식베이스 영향도 분석 수행
- 이미지 요약 설명 추가, 키워드 메타 보강, 질문-문서 분류기 검토
- Hybrid Search와 chunk weighting 제외 실험 수행
- 미매칭 원인 분석과 Reference 재검증 수행
- SFT 포함 평가 및 튜닝 전후 비교 수행

### 체크리스트

- [ ] 미매칭 22.5%를 문서 미검색과 문서 매칭 오류로 구분해 정리합니다.
- [ ] Reference 오기입과 정답지 누락 여부를 별도로 검수합니다.
- [ ] 검색 개선이 필요한 문제와 답변 생성 품질과 무관한 문제를 분리합니다.
- [ ] PC 카테고리를 SFT 평가에 포함했을 때 실제 개선이 있는지 재측정합니다.

## 상담스크립트

### 누적 수행 내용

- 기본 데이터 분석 수행
- Query Expansion과 Workflow 분리 테스트 수행
- BM25 기반 접근과 분류 문제 전환 검토
- 스크립트 목적 자동 생성 및 Rule-based 분류 테스트 수행
- 매칭 성공 케이스의 실제 정답 여부 분석 수행
- 골든셋 수동 검수와 청크 ID 매핑 진행

### 체크리스트

- [ ] 스크립트 목적 분류 방식의 운영 가능성을 검토합니다.
- [ ] 문서 매칭 성공과 실제 답변 정답 여부를 분리해서 측정합니다.
- [ ] CS 골든셋 청크 ID 미완성 건을 모두 정리합니다.
- [ ] 검색 구조 개선보다 분류형 접근이 유리한지 결론을 낸습니다.

## 상품매뉴얼

### 누적 수행 내용

- 기본 데이터 분석 수행
- Hallucination 억제 프롬프트 실험 수행
- 상품명 기반 Routing과 상품명 추출 검색 실험 수행
- 형태소 분석 기반 BM25 검색과 Hybrid 검색 시도
- 검색 실패 원인 분석 및 튜닝 전후 비교 수행
- 골든셋 수동 검수 진행

### 체크리스트

- [ ] 상품명 추출 기반 검색의 재현성과 fallback 전략을 확정합니다.
- [ ] 검색 실패와 답변 생성 오류를 분리해 케이스북을 만듭니다.
- [ ] 상품 조건/특약 구조 학습이 실제 정답률 개선으로 이어지는지 확인합니다.
- [ ] BM 카테고리의 튜닝 회귀 원인을 데이터, 검색, 생성 단계로 나눠 분석합니다.

- [ ] 검색 실패와 생성 실패 비중을 수치로 정리합니다.
- [ ] 튜닝 전후 비교 테스트를 실제 실행하고 결과를 남깁니다.

## 약관

### 누적 수행 내용

- 상품 간 약관 유사도 분석 수행
- Hallucination 억제 프롬프트와 기본 프롬프트 개선 검토
- 문서는 찾았지만 답변이 틀리는 케이스 집중 분석
- 튜닝 전후 비교와 골든셋 수동 검수 수행
- 색인 미등록, 청크 오매핑 등 데이터 품질 이슈 확인

### 체크리스트

- [ ] 약관 답변은 검색 성공 후 생성 실패를 별도 핵심 지표로 관리합니다.
- [ ] 법률 용어와 약관 구조를 반영한 생성 프롬프트 개선안을 확정합니다.
- [ ] 색인 미등록 상품과 청크 오매핑 이슈를 우선 수정합니다.
- [ ] 튜닝 회귀 원인이 데이터 문제인지 모델 문제인지 분리해 판단합니다.
- [ ] API 입력값에 약관 ID를 함께 전달하는 외부 협의 상태를 추적합니다.
