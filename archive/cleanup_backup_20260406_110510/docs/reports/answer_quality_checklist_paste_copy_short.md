### 핵심 메시지

## 슬라이드 3. 전체 현황

### 제목
- 체크리스트 전체 현황

### 핵심 메시지
- 큰 방향은 이미 착수됐지만, 높음 우선순위 미착수 5건이 핵심 공백입니다.

### 본문 3줄
- 전체 항목 34건 중 25건 진행중, 9건 미착수
- 높음 우선순위 항목은 24건, 이 중 5건은 아직 미착수
- 지금 필요한 것은 신규 발굴보다 우선순위 공백 해소

### 표 텍스트
| 구분 | 건수 | 해석 |
|---|---:|---|
| 전체 항목 | 34 | 전체 관리 대상 |
| 진행중 | 25 | 핵심 축은 이미 착수 |
| 미착수 | 9 | 남은 공백 존재 |
| 높음+미착수 | 5 | 단기 최우선 실행 대상 |

### 차트 텍스트
- 도넛: 진행중 25 / 미착수 9
- 막대: 높음 24 / 중간 10

## 슬라이드 4. Learn 1

### 제목
- Learn 1. 데이터와 평가 기준이 흔들리면 결과 해석도 흔들린다

### 핵심 메시지
- 품질 개선의 출발점은 모델이 아니라 평가 기반의 신뢰도습니다.

### 본문 3줄
- 골든셋, 정답지, Chunk ID 정합성이 확보돼야 평가 결과를 신뢰할 수 있음
- 수동 검수와 자동 평가 결과를 함께 기록해야 해석 왜곡을 줄일 수 있음
- 기준 데이터가 흔들리면 검색 개선과 모델 개선 효과를 구분할 수 없음

### 도식 텍스트
- 하단: 데이터 정합성
- 중단: 평가 기준 통일
- 상단: 실험 결과 해석

---

## 슬라이드 5. Learn 2

### 제목
- Learn 2. 검색 성공과 생성 성공은 별도로 관리해야 한다

### 핵심 메시지
- 검색은 문서를 찾는 문제이고, 생성은 근거로 답을 만드는 문제입니다.

### 본문 3줄
- Hybrid Search, BM25, Reranker, Top-K 조합은 검색 품질에 직접 영향
- 정답 문서를 찾아도 hallucination, 근거 누락, 표현 왜곡은 별도 발생 가능
- 검색 실패와 생성 실패를 분리 측정해야 개선 원인을 정확히 해석할 수 있음

### 도식 텍스트
- Step 1: Query -> Retrieval -> Relevant Chunk
- Step 2: Context -> Generation -> Grounded Answer
- 실패 구분: 검색 실패 / 생성 실패

## 슬라이드 6. Learn 3

### 제목
- Learn 3. 많이 넣는 것보다 정확하게 넣는 것이 중요하다

### 핵심 메시지
- 컨텍스트 수량보다 컨텍스트 정제와 워크플로우 설계가 더 중요합니다.

### 본문 3줄
- Context Refinement와 상위 컨텍스트 선별이 노이즈 감소에 효과적임
- Self-Verification, Self-Consistency 같은 후처리 전략 비교가 필요함
- 문서 간 오염과 상호 영향도까지 봐야 grounded answer 품질이 안정화됨

### 도식 텍스트
- 비교 1: 많은 컨텍스트 입력
- 비교 2: 정제된 컨텍스트 입력
- 비교 축: 품질 / 지연시간 / 노이즈

## 슬라이드 7. Do

### 제목
- Do. 즉시 실행 과제

### 핵심 메시지
- 남은 공백은 개념 논의가 아니라 실행 단위로 쪼개서 바로 관리해야 합니다.

### 본문 3줄
- 학습·검증 세트를 최신화하고 정답지와 Chunk 매핑을 보강
- 검색 실패 코드북과 생성 실패 케이스북을 분리 운영
- 운영 반영 전 회귀 케이스와 부작용 점검 체계를 정착

### 표 텍스트
| 우선 과제 | 핵심 액션 | 기대 효과 |
|---|---|---|
| 평가 기반 정비 | 검증 세트 최신화, 정답지 정합성 확보 | 평가 신뢰도 확보 |
| 검색 구조 개선 | Hybrid Search/BM25/Reranker 최적화 | 회수율 향상 |
| 생성 안정화 | hallucination 억제, 출처 흐름 강화 | grounded answer 강화 |
| 운영 통제 | 회귀/부작용/주간 기록 체계화 | 배포 리스크 감소 |
